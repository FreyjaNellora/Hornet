# EXP-023 — bootstrap corpus regeneration (B5): clean ordering + decisive config

- **Date:** 2026-06-10 · **Status:** running
- **Goal:** replace the tainted 133-game bootstrap corpus. The original (EXP-013) was generated on
  the **maxn path at beam 4 with the inverted free-capture heuristic live** — EXP-020 measured
  that bug changing **11.6% of played moves** at beam 4 — and was **drawish** (150-ply cap, few
  eliminations → weak "points-ahead-at-cap" outcome labels for Texel).
- **Lever / change:** `examples/bootstrap.rs` regeneration config (a data-quality decision,
  recorded here; not a shipped play default).

Old corpus preserved in git history (commit 6a2b6a9 and earlier); working tree regenerated
in place so `texel_tune` keeps pointing at `selfplay_games/`.

**Execution note:** a single instance measured ~12 min/game at this config (the flashlight at cap
1200 evaluates every kept node's children at each of 8 levels — ~30 h for 150 games,
intractable single-threaded), so `bootstrap` gained a `[N] [START]` range form and the corpus is
generated by **five parallel instances of 30 games each** (games 0–29, 30–59, …, 120–149; seeds
derive from the game index, so ranges are collision-free; 6-core box, ~6–8 h wall expected).
